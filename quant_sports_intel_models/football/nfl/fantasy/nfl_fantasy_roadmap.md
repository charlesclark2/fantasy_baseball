## 1. Strategic frame
- 🥇 **REDRAFT IS THE PRIORITY (operator 2026-07-22): redraft is the MAJORITY of fantasy-football leagues; dynasty is far less prevalent.** So NF1 (seasonal/weekly redraft projections) + the redraft TOOLS lead; **Dynasty (NF2) is the underserved *differentiator* but SECONDARY by league prevalence** — build it, but after the redraft product.
- **NFL fantasy is the LARGEST fantasy market.** The betting product's efficient-market caveat does NOT apply; calibrated projections are pure product value. **Honest-uncertainty framing** throughout (credible intervals, not false-precise points).
- 🎯 **THE PRODUCT WE'RE COMPETING WITH = "The Fantasy Footballers"** (operator benchmark) — rankings + start/sit + trade + draft tools. ⇒ the bar is a full TOOLS suite on top of the projections, not just projections.
- 🧱 **THE KEYSTONE ARCHITECTURE — a LEAGUE-CONFIG / SCORING layer (NF-C1):** every tool (draft optimizer, trade calc, FA rankings, start/sit) must be **CUSTOMIZABLE BY LEAGUE** — so NF1's projections are RAW stat lines, and a league-config layer (scoring rules + roster/starter slots + league size) converts them into league-specific fantasy points / values / VOR. Build the config layer ONCE; every tool reads it. ⭐ **This engine is SHARED with the MLB fantasy vertical (`baseball/fantasy/`) — build the league-config + tool ENGINES once, instantiate per sport** (the `hierarchical.py`-style reuse; the projection INPUTS differ, the optimizer/trade/config LOGIC doesn't).
- 🔌 **PLATFORM INTEGRATION IS THE HARD PREREQUISITE (NF-C0) — you can't customize by league without INGESTING the user's league** (rosters + settings + live draft state). **Football platforms, by 2025 MAU (Sensor Tower Aug-2025): ESPN 48% · Sleeper 33% (fastest-growing, dynasty-heavy) · Yahoo 18%** — those THREE cover ~99% of redraft; then **CBS Sports** (mid) + **MyFantasyLeague (MFL)** & **Fantrax** (the serious-DYNASTY platforms) + **NFL.com Fantasy** (smaller). ⭐ **API reality drives the build order (verify current status per platform): Sleeper (public read-only API, well-documented — easiest × #2 reach) + Yahoo (official OAuth API) are the clean high-value STARTS; ESPN is the must-have but its API is UNOFFICIAL/cookie-based/fragile; MFL is API-friendly (the dynasty long-tail); CBS/Fantrax have APIs; NFL.com is limited.** ⇒ integration order ≈ **Sleeper + Yahoo → ESPN → CBS/MFL/Fantrax.**
- ⏰ **GTM TIMING — draft season is NOW (Aug–early Sept; the operator's own draft 8/22):** a working draft optimizer + league import for the top platforms during draft season is a PRIME GTM push — a user imports their real ESPN/Sleeper/Yahoo league and drafts with our tool. ⇒ the near-term MVP = NF-C0 (Sleeper+Yahoo+ESPN import) + NF-C1 (config) + NF-C2 (draft optimizer) + NF-FASTPATH projections, shippable for the draft window.

## 2. Phased plan (NF-series; the shared core in §0 is the input)
- **NF1 — WEEKLY + SEASON FANTASY-POINT PROJECTIONS** (the core fantasy model): from the shared N1.2 player-week projection → fantasy points (std + PPR — the validated N0.3 PPR calc) via opportunity × efficiency × role (`dim_player_role`). §0.5 where a model is selected; posterior-predictive; honest uncertainty; validated vs a holdout season.
- **NF2 — DYNASTY / ROOKIE BOARD (⚠️ SECONDARY by league prevalence — redraft is the majority; build after the redraft product + tools):** rank players/prospects for a multi-year hold; ⭐ the rookie leg consumes NCAAF-P1A as a **RESIDUAL ON DRAFT SLOT** (rank on draft position + where `projected_nfl_z` disagrees; skill positions RB/TE/QB/WR only; ≈0 for DL/LB/DB; recalibrate the parameter uncertainty). The underserved differentiator.

**THE TOOLS LAYER (the Fantasy-Footballers-competitive suite; league-customizable; the ENGINE is SHARED with `baseball/fantasy/` — build once, instantiate per sport):**
- **NF-C1 — ⭐ LEAGUE CONFIGURATION + SCORING ENGINE (the KEYSTONE — build FIRST of the tools):** a league-settings schema (scoring rules per stat, roster/starter slots, league size, PPR variant, superflex…) + a scorer that converts NF1's RAW stat-line projections → **league-specific fantasy points + VOR/value + positional scarcity**. Every tool below reads it.
- **NF-C2 — DRAFT OPTIMIZER (⭐ LIVE-draft-aware):** given the league config + the current DRAFT STATE (who's gone, my roster, my slot) → the value-maximizing pick (VOR + positional-need + tier-break aware), **usable DURING a live draft** (real-time state input). The flagship draft tool.
- **NF-C3 — TRADE CALCULATOR (redraft ROS-value + dynasty multi-year):** value a proposed trade for MY league config — redraft = ROS value; dynasty = multi-year (consumes NF2/keeper). Honest "who wins + by how much + uncertainty."
- **NF-C4 — FREE-AGENT / WAIVER RANKINGS (customizable by league ROSTERS):** rank available players by league-adjusted ROS value RELATIVE to a given roster's needs (roster-aware, not a generic list). The weekly in-season churn tool.
- **NF3 — FANTASY APP SURFACES:** the projections + ALL the tools (draft optimizer, trade calc, FA rankings, start/sit) + the dynasty board in `frontend/` (Next.js; app-target guard). Honest-uncertainty display.
- ⏰ **NF-FASTPATH (near-term, does NOT need the full NF1 model): 2026 SEASON projections by 8/22 (operator's draft).** `mart_projections_preseason` (N0.3, dynamic) already gives a first-pass 2026 VETERAN projection with light validation/ranking; rookies attach when P1A is available. Ship for 8/22; NF1 refines after. Do NOT delay NCAAF for this (P1A is a NCAAF story serving both).
