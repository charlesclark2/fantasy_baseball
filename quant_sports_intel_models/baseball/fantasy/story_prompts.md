# Fantasy & Dynasty — Per-Story Prompts (standalone sessions)

**What this is:** one **`▶ Story prompt`** per *not-yet-completed* story in the Fantasy/Dynasty vertical (F1–F8), so any card can be pulled and run in a fresh Claude Code session on its own. Companion to **`fantasy_dynasty_guide.md`** (the F-stories' full tasks/AC + the §0.2 data contract) and the program's per-story convention (`../edge_program/edge_program_implementation_guide.md` §0.4).
**Last updated:** 2026-06-18 _(refresh on any material change)_

**How to use:** copy a story's block into a fresh session. Compact by design — blocks inherit shared context by reference. All blocks assume the inherited conventions: `dbtf` (not `dbt`); Snowflake via the Snowflake MCP, fully-qualified, no `USE`; `uv run python`; hand >1-min scripts to the operator; **do not git commit/push**; **honest framing** — the bar is *match or beat ZiPS/Steamer* on transparent accuracy, never a betting win-rate or "we're best."

**Cross-vertical dep:** **F4 depends on Edge `E7`** (minor-league MLEs). App surfaces marked **⏳ §0.3** are generated by the upstream model session, not hand-authored.

**Status legend:** ⬜ open · 🔄 in progress · 🔒 gated · ✅ done.
